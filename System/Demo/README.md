# Dex Demo Workspace

This folder contains a Codex-ready sample Dex workspace.

Use it when you want to explore the system without mixing demo content into a real vault.

## How to use it

1. Start Codex in the repo root.
2. Ask Codex to inspect `System/Demo/`.
3. Use the sample people, companies, meetings, and backlog files to understand the Dex structure.

## What this is

- `System/Demo/Inbox/` contains sample meeting and planning inputs
- `System/Demo/People/` and `System/Demo/Areas/` show relationship memory structure
- `System/Demo/System/` shows supporting Dex system files

## What this is not

- It is not the primary setup path for a real install.
- It is not the active Codex runtime surface.
- It should not replace onboarding in a real workspace.

For real setup instructions, use the repo root `README.md`.
